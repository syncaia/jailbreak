# Security Policy

## Supported Versions

We strive to keep our jailbreak methods and tools up-to-date with the latest verified exploits. Currently, security updates and patches are provided for the following versions:

| Version | Supported          | Notes                                     |
| ------- | ------------------ | ----------------------------------------- |
| Latest  | :white_check_mark: | Active development and exploit testing.   |
| Older   | :x:                | We do not provide patches for older methods. |

## Reporting a Vulnerability

If you discover a security vulnerability within this project (e.g., malicious scripts inadvertently included in the resources, or an exploit that can be abused beyond the scope of a standard jailbreak), please report it responsibly.

**Do NOT open a public issue for security vulnerabilities.**

Instead, please send an email directly to the project maintainers (or use GitHub's private vulnerability reporting feature if enabled on this repository).

We will try to acknowledge your report within 48 hours and provide an estimated timeline for resolution. Public disclosure should only happen after a fix has been successfully implemented and merged into the main branch.
